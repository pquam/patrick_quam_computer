The behind the scenes and usecase of this website:

The stack

I made this website completly from the ground up. There is a local beta server I designed and built so that I can test changes wihtout messing up the production website. the local nginx webserver is being hosted in a VMWare virtual machine, while the production webserver is being hosted by a VPS. I wrote all the HTML, CSS, and Javascript for the website in VSCode and VIM. I manage the production and beta branches using the git commandline interface & github. The website uses CERTBOT and cronjobs to automate SSL certificate renewal so that the website can support HTTPS.

What is it for?

This website is intended mostly for my personal use to keep track of notes, as well as a place for me to practice implementing technology I find interesting. In the process of setting up this website, I have learned a ton about linux, how servers work, how DNS works, freshened up my HTML and CSS skills, and acheived a more complete understanding of git and github. While I continue to refine the website and expand its scope, I hope to learn more about Javascript and other web development platforms.
