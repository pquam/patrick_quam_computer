Setting Up User Accounts and Running NGINX

User Account Setup

  1. Create a root user that is set to nologin and cannot be accessed via SSH. This is the default behavior for the root user on many systems.
  2. Create a personal user account with sudo permissions. This account can be accessed via SSH, and you can use sudo to perform administrative tasks.
  3. Create a nginx user account with minimal permissions. This account should also be set to nologin and should not be accessible via SSH.

Running NGINX

  1. Install NGINX using your personal account with sudo permissions.
  2. In the NGINX configuration file, specify that NGINX should run as the nginx user. This is typically done with the user directive in the main context of the configuration file. For example, you might have a line like this in your nginx.conf:

user nginx;
  1. Start the NGINX service (usually with a command like sudo systemctl start nginx). NGINX will start with root privileges, but then drop its privileges and run as the nginx user.

In this setup, there's no need to use setuid or sudo -u. NGINX handles the privilege dropping itself.
